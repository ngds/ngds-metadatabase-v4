User Components

Saved Sets are made up of zero or more saved searches (which identify a group of citations) and/or additional identified citation records. Saved Sets are created in this tab, then populated by saving searches and citations on the Search page. Saved Sets can be used as the source for the creation of "Test Sets", which are a data product identified in the GeoDeepDive system used in ML processing. The Request Test Set button sends the Saved Set data to GeodeepDive for processing. When the Test Set is complete, GeoDeepDive system returns the results, which then appears in the Test Set tab.


Dictionaries are a collection of search terms, and can be either Geodeepdive identified Dictionaries, or user constructed dictionaries. Dictionaries can also be used as the source for the creation of "Test Sets", which are a data product identified in the GeoDeepDive system used in ML processing. The Request Test Set button sends the Saved Set data to GeodeepDive for processing. When the Test Set is complete, GeoDeepDive system returns the results, which then appears in the Test Set tab.


Test Sets are data products (usually the full-text of a document) provided by the GeoDeepDive System. Test sets can only be requested by authorized users and are identified by GDD API Keys. Test sets are made up of text samples from various citations that conform to a specific search criteria (a dictionary or saved set). These test documents are meant to aid in application development by giving users a broad, random sample of full-text documents that meet their search criteria.
The Sync with CyVerse button will automatically send the test-set information (the api-key and test-set name) to a userâ€™s CyVerse data store. From the CyVerse Data store, the API key can easily be used to download the test-sets directly into into CyVerse Applications (such as RStudio or Jupyter Notebooks), where the user can experiment with developing a full-text analysis application.

Upload Document provides capability to allow authorized users to upload documents (citations) to the GeoDeepDive system.


Status provides a transaction history and current state of all requests and processes in the Adept, GeoDeepDive, Cyverse, and COSMOS Systems.
